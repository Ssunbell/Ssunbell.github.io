## 배운 것들

1. git checkout으로 head를 컨트롤 할 수 있음
2. git checkout master를 하지 않으면 head가 master를 가리키지 않으므로 커밋이 master와 따로 생성됨
3. git reflag를 통해 모든 나의 행동들을 다시 볼 수 있음

## 느낀점
1. git에 대해 잘 모르는 점들이 많았는데 알던 것들을 확인하고 새로운 것들을 배워서 좋았음.
